# تشغيل Barakat Help Desk على Hostinger Hosting

هذه النسخة مخصصة للاستضافة العادية في Hostinger، وليست VPS. تعمل بـ PHP + MySQL داخل مجلد `public_html`.

## 1. إنشاء قاعدة البيانات من Hostinger

1. افتح Hostinger hPanel.
2. ادخل إلى **Databases** ثم **MySQL Databases**.
3. أنشئ قاعدة بيانات جديدة.
4. أنشئ مستخدمًا لقاعدة البيانات وكلمة مرور قوية.
5. اربط المستخدم بقاعدة البيانات بصلاحيات كاملة.
6. احتفظ بهذه البيانات:
   - Database Host
   - Database Name
   - Database User
   - Database Password

غالبًا يكون Database Host هو `localhost`، لكن استخدم القيمة التي تظهر لك في Hostinger.

## 2. تعديل ملف الاتصال

داخل مجلد `public_html` افتح ملف `config.php` واكتب بيانات قاعدة البيانات:

```php
const DB_HOST = 'localhost';
const DB_NAME = 'اسم_قاعدة_البيانات';
const DB_USER = 'اسم_مستخدم_قاعدة_البيانات';
const DB_PASS = 'كلمة_مرور_قاعدة_البيانات';
```

لا تضع كلمة مرور لوحة Hostinger هنا. المطلوب كلمة مرور مستخدم قاعدة البيانات فقط.

## 3. رفع الملفات

1. افتح **File Manager** في Hostinger.
2. افتح مجلد `public_html`.
3. ارفع محتويات مجلد `hostinger/public_html` بالكامل إلى `public_html`.
4. تأكد أن الملفات مثل `index.php` و `api.php` و `config.php` موجودة مباشرة داخل `public_html`، وليس داخل مجلد إضافي.

## 4. تشغيل النظام لأول مرة

1. افتح رابط الدومين أو الرابط المؤقت من Hostinger.
2. إذا كانت قاعدة البيانات جديدة، سيحولك النظام تلقائيًا إلى `/setup`.
3. أنشئ أول حساب مدير للبرنامج.
4. بعد الدخول، افتح قسم **الأشخاص** لإضافة الموظفين وأعضاء الفريق وتحديد الصلاحيات.

## 5. الصلاحيات

- مدير البرنامج: إضافة المستخدمين، إدارة الطلبات، التقارير.
- مشرف فريق: إدارة الطلبات وإضافة المستخدمين.
- عضو فريق: استلام الطلبات وتحديث الحالات وإغلاق الطلبات.
- موظف: إنشاء الطلبات ومتابعتها وتأكيد الاستلام أو إعادة فتح الطلب.

## 6. ملاحظات مهمة

- مجلد `uploads` يجب أن يظل موجودًا داخل `public_html`.
- لا تحذف ملف `.htaccess` لأنه مسؤول عن تشغيل روابط `/login` و `/setup` و `/api`.
- عند اكتمال الطلب، يصل إشعار مباشر داخل النظام لصاحب الطلب.
- التقارير الأسبوعية يتم تنزيلها من قسم التقارير داخل لوحة التحكم.
