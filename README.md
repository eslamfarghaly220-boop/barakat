# Barakat Help Desk

نظام طلبات داخلي لقسم تقنية المعلومات والتصميم، أقرب إلى Help Desk وخدمة طلبات داخلية وليس تطبيق إدارة مهام عادي.

## المزايا

- تسجيل دخول وصلاحيات: مدير البرنامج، مشرف فريق، عضو فريق، موظف.
- إنشاء طلبات: دعم تقني، تصميم، صلاحيات، جهاز، بريد إلكتروني، تعديل موقع، وأخرى.
- مرفقات للطلب وملف نهائي عند الإغلاق.
- رقم طلب تلقائي وحالات واضحة: جديد، قيد المراجعة، جاري التنفيذ، مكتمل، تم تأكيد الاستلام، معاد فتحه.
- إشعار مباشر لطالب الطلب عند اكتمال طلبه، مع متابعة تلقائية للإشعارات داخل الواجهة.
- إسناد الطلب لمسؤول من الفريق.
- لوحة تحكم للطلبات الجديدة والمتأخرة والمكتملة.
- تقارير أداء للفريق ومتوسط مدة الإنجاز.
- تنزيل تقارير أسبوعية للطلبات المنجزة والمتأخرة.
- إدارة الأشخاص من مدير البرنامج أو المشرف.
- قاعدة بيانات SQLite داخلية مع دعم مسار تخزين ثابت للإنتاج.

## التشغيل المحلي

```bash
npm start
```

على Windows إذا منع PowerShell تشغيل npm، استخدم:

```powershell
cmd /c npm start
```

ثم افتح:

```text
http://localhost:3000
```

## أول حساب مدير

عند أول تشغيل على قاعدة بيانات جديدة، ينشئ النظام حساب مدير برنامج واحد فقط. اضبط بياناته قبل أول تشغيل إنتاجي من خلال متغيرات التشغيل:

```text
INITIAL_ADMIN_NAME
INITIAL_ADMIN_EMAIL
INITIAL_ADMIN_PASSWORD
INITIAL_ADMIN_DEPARTMENT
```

إذا لم يتم تحديد كلمة المرور، سينشئ النظام كلمة مرور مؤقتة ويعرضها في سجل التشغيل. بعد الدخول، يمكن لمدير البرنامج إضافة باقي الأشخاص من داخل النظام.

## قاعدة البيانات والمرفقات

- محليًا: قاعدة البيانات في `data/helpdesk.sqlite` والمرفقات في `uploads/`.
- للإنتاج: يمكن ضبط `STORAGE_DIR` حتى تكون قاعدة البيانات والمرفقات على مساحة تخزين ثابتة.
- يمكن ضبط `DATA_DIR` و `UPLOAD_DIR` بشكل منفصل عند الحاجة.
- ملفات قاعدة البيانات والمرفقات غير مرفوعة إلى GitHub لحماية البيانات.

## النشر

المشروع يحتوي على:

- `Dockerfile` لتشغيل التطبيق على أي استضافة تدعم Docker.
- `render.yaml` كنموذج نشر على Render مع مساحة تخزين ثابتة.
- `.github/workflows/ci.yml` لفحص الكود عند الرفع على GitHub.
- `.env.example` يوضح إعدادات التشغيل المطلوبة.

مهم: GitHub يحفظ الكود فقط. تشغيل النظام بقاعدة بيانات ومرفقات يحتاج استضافة مثل Render أو Railway أو VPS.
